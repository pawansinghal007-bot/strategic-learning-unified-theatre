# Missing Testability Hooks Report

## Prioritized Recommendations

| Priority | Recommendation | Source Files | Why It Is Needed | Effort |
| ---: | --- | --- | --- | --- |
| 1 | Add stable `data-testid` hooks and accessible names to all critical dashboard controls and status surfaces. | `electron-ui/src/**`, `renderer/**`, dashboard components that render accounts, browser pane, workspace policy, approvals, audit, quotas, routing, and shell/header UI | Page objects need durable selectors for account switching, capture status, policy forms, approval resolution, audit export, quota modes, routing rows, and daemon health. Without these, tests fall back to brittle text/CSS selectors. | Medium |
| 2 | Add centralized IPC invocation telemetry for start, end, duration, status, and errors. | `electron-ui/ipc/handlers.cjs`, `electron-ui/ipc/*-handlers.cjs`, `electron-ui/main.cjs`, `src/main/ipc/ipcAdapter.ts` | Smoke/regression/full tests need reliable evidence for preload/API readiness, IPC failures, latency, and rejected calls. This also makes artifact logs actionable in CI. | Medium |
| 3 | Emit deterministic browser capture lifecycle events with response id, platform, content length, selector source, and failure reason. | `electron-ui/preload-browser.cjs`, `electron-ui/browser-pane.cjs`, `electron-ui/main.cjs`, `electron-ui/ipc/capture-handlers.cjs` | Capture tests currently depend on timing-sensitive DOM behavior. Explicit lifecycle events allow tests to wait for capture success, timeout, selector fallback, unknown platform, and failed capture states. | Medium |
| 4 | Move AI platform selectors behind a versioned selector config with E2E override support. | `electron-ui/preload-browser.cjs`, `electron-ui/browser-pane.cjs`, `src/internal/config.js` | Full-suite selector fallback tests need deterministic local fixture selectors. External platform DOM selectors should be overridable by `$HOME/.vscode-rotator/browser-selectors.json` without code changes. | Medium |
| 5 | Add routing decision event records with candidates considered, selected provider, skipped providers, policy source, quota status, and reason code. | `electron-ui/ipc/workspace-routing-handlers.cjs`, `src/policies/**`, routing/gateway modules under `src/**` | J2, J4, and J5 require explainable routing assertions across UI, preload API, persisted JSON, and restart. Reason strings alone are too fragile; tests need structured reason codes. | High |
| 6 | Add audit event schemas and stable event type constants for policy, approval, quota, routing, capture, and account-switch actions. | `src/audit/audit-log.ts`, `electron-ui/ipc/audit-handlers.cjs`, `src/governance/**`, `src/policies/**`, `src/accounts/switcher.js` | Audit assertions should verify typed events and linkage ids rather than text fragments. This is required for approval linkage, quota exceeded evidence, routing blocked evidence, and tamper scenarios. | Medium |
| 7 | Add approval state transition hooks and idempotency metadata. | `src/governance/workspace-approvals.ts`, `electron-ui/ipc/workspace-approval-handlers.cjs`, approval UI components | Regression/full tests need to assert requested, approved, rejected, cancelled, expired, already-resolved, and unknown-id behavior without racing policy application. | Medium |
| 8 | Add quota evaluation diagnostics with clock source, daily/weekly buckets, mode, boundary state, and resulting action. | `src/governance/workspace-quotas.ts`, `electron-ui/ipc/workspace-policy-handlers.cjs`, quota UI components | Quota boundary, reset, alert, fallback, block, and notification-isolation tests need deterministic state that is not inferred from display text or wall-clock timing. | Medium |
| 9 | Add test-mode controls for fake VS Code restart and auth target validation. | `src/accounts/switcher.js`, `src/vscode.js`, `electron-ui/ipc/handlers.cjs` | Account-switch E2E must prove auth handoff without launching real VS Code. A first-class fake restart/auth-target mode reduces platform flake and avoids accidental writes outside isolated `HOME`. | Medium |
| 10 | Add daemon and storage monitor lifecycle events plus test-owned PID/lock reporting. | `electron-ui/main.cjs`, `src/daemon/daemon-runner.js`, daemon/watch/storage modules under `src/**` | Full-suite daemon tests need deterministic waits for stale PID cleanup, scheduled capture/enhancement attempts, scan completion, partial failures, and teardown. | High |
| 11 | Add corruption recovery result objects for governance JSON stores. | `src/policies/workspace-policy.ts`, `src/governance/workspace-approvals.ts`, `src/governance/workspace-quotas.ts`, routing history modules, `src/audit/audit-log.ts` | Full-suite corruption tests need scoped error/fallback results per store and assurance that unrelated stores were not reset. | High |
| 12 | Add export result metadata for audit and routing exports. | `electron-ui/ipc/audit-handlers.cjs`, `electron-ui/ipc/workspace-routing-handlers.cjs`, `electron-ui/ipc/workspace-report-handlers.cjs` | Export tests should assert path, format, event count, filter summary, integrity status, and failure state without parsing UI toast text. | Low |
| 13 | Add renderer-visible status hooks for feature flag enforcement. | `src/internal/config.js`, `electron-ui/ipc/handlers.cjs`, feature-guarded IPC handlers, dashboard feature panels | Full-suite fail-closed tests need clear UI/API evidence that disabled `localDbEnabled`, `browserCaptureEnabled`, or `llmCommandsEnabled` blocked only the intended operation. | Low |
| 14 | Add prompt library validation result codes. | `electron-ui/ipc/handlers.cjs`, prompt library modules under `src/**` or `electron-ui/**` | Full-suite prompt tests need stable errors for invalid JSON, duplicate ids, blank prompts, max length, and over-limit content. | Low |
| 15 | Add CI artifact log sinks for renderer console, main-process console, mutated store snapshots, daemon logs, and seeded metadata. | `e2e/support/electron.ts`, `e2e/support/reset.ts`, `e2e/support/env.ts`, `playwright.config.ts` | The CI design requires rich artifacts beyond screenshots/traces/videos. Support code needs standard collection points and naming so failures are reviewable. | Medium |

## File-Level Hook Checklist

| Area | Files | Hook Changes |
| --- | --- | --- |
| Dashboard selectors | `electron-ui/src/**`, `renderer/**` | Add `data-testid` to tabs, panels, lists, rows, forms, buttons, meters, badges, and export controls named in `04a-selectors-and-hooks.md`. |
| Main process IPC and lifecycle | `electron-ui/main.cjs`, `electron-ui/ipc/handlers.cjs`, `electron-ui/ipc/*-handlers.cjs` | Wrap every `ipcMain.handle` path with structured telemetry and sanitized error payloads. |
| Contract IPC | `src/main/ipc/ipcAdapter.ts`, `src/shared/ipc/contract.ts` | Ensure contract-channel responses include stable status codes, correlation ids, and validation errors. |
| Browser capture | `electron-ui/preload-browser.cjs`, `electron-ui/browser-pane.cjs`, `electron-ui/ipc/capture-handlers.cjs` | Add selector override loading, capture lifecycle events, unknown-platform status, timeout status, and content-length metadata. |
| Account switching | `src/accounts/switcher.js`, `src/accounts/store.js`, `src/accounts/secret-store.js`, `src/vscode.js` | Add test-only fake restart mode, auth-target verification metadata, secret-read failure codes, and non-secret error surfaces. |
| Workspace policy/context | `src/policies/workspace-policy.ts`, `electron-ui/ipc/workspace-policy-handlers.cjs` | Return policy source, merge trace, validation result, approval requirement, and enterprise restriction reason. |
| Approvals | `src/governance/workspace-approvals.ts`, `electron-ui/ipc/workspace-approval-handlers.cjs` | Return transition result, immutable requester metadata, resolution metadata, and duplicate/expired/unknown-id reason codes. |
| Quotas | `src/governance/workspace-quotas.ts`, `electron-ui/ipc/workspace-policy-handlers.cjs` | Return bucket math, mode, action, boundary state, notification id, and routing impact metadata. |
| Routing | `electron-ui/ipc/workspace-routing-handlers.cjs`, routing/gateway modules under `src/**` | Persist structured explanations with provider candidates, skip reasons, quota context, policy source, and deterministic ordering keys. |
| Audit | `src/audit/audit-log.ts`, `electron-ui/ipc/audit-handlers.cjs` | Add event schema version, event type constants, correlation ids, verification diagnostics, tamper reason, and export result metadata. |
| Daemon/storage monitor | `electron-ui/main.cjs`, `src/daemon/daemon-runner.js`, storage monitor modules under `src/**` | Emit scheduled task start/end/failure events, scan summaries, stale PID/lock cleanup events, and teardown confirmation. |
| E2E support | `e2e/support/**`, `playwright.config.ts` | Standardize console collection, store snapshots, seeded metadata, artifact names, and test-owned process cleanup. |
