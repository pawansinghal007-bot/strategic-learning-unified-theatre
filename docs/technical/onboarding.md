# Onboarding

## Getting started with the docs
- Start with `docs/README.md` to understand the structure and intended audience.
- Read `docs/product/overview.md` for the product vision.
- Read `docs/technical/architecture.md` and `docs/technical/system-context.md` for repo context.

## Using the repo
- The project root is `E:\VS Code Agent\Solution`.
- Use Node 18+ and run tests with `npm test` in the `Solution` folder.
- No build step is required for the core repo.

## Documentation workflow
- Keep active sprint state in the DB.
- Keep execution rules in `strategic-learning-unified-theatre-master-instructions.md`.
- Update curated docs at sprint close when durable outcomes change.
