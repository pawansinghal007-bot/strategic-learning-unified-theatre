# Documentation History

## Purpose
Keep a concise audit trail of how the documentation system evolved.

## Current history
- Final documentation system adopted with audience-based sections under `docs/`.
- Existing starter docs migrated into `docs/product/`, `docs/technical/`, `docs/operations/`, `docs/security/`, and `docs/marketing/`.
- `docs/operations/documentation-policy.md` was added to define ownership and maintenance rules.
- `docs/README.md` was created to describe the overall docs structure.

## When to archive
- When a doc is superseded by a newer reference.
- When an operational process changes and the old version must be preserved.
- When launch messaging or positioning is intentionally retired.

## Guidelines
- Archive only durable history, not active sprint state.
- Keep archived files readable and dated.
- Use `docs/archive/` for historical references, not the main docs navigation.
