# Documentation Archive

This folder keeps historical planning notes, sprint prompts, blueprints, and imported reference material that are not part of the current operating documentation.

Current documentation should live at the top of `docs/` or in an active subfolder such as `docs/technical/`. Historical source material belongs here so the project root stays focused on runnable code and current docs.

## Folders

- `blueprints/` - Historical implementation blueprints.
- `external/` - Material consolidated from obsolete sibling folders.
- `historical/` - Older summaries and completion notes retained for traceability.
- `prompts/` - Historical sprint prompt files.
- `sprints/` - Older sprint confirmation and sprint-specific planning notes.
