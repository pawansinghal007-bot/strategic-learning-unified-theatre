# FAQ

## Why is the docs system split across folders?
The structure separates product, technical, operations, security, and marketing content so readers can find stable reference material quickly.

## Where do I put sprint state?
Sprint state belongs in the DB and master instructions, not in curated docs.

## Can I edit the docs during a sprint?
Yes, but only for durable reference items. Temporary sprint notes should stay in the database or in work-in-progress drafts, not in curated docs.

## What should I update at sprint close?
- `docs/README.md` if the docs structure or process changes.
- `docs/operations/documentation-policy.md` for new doc ownership or maintenance rules.
- Relevant `docs/*` files when durable product, technical, or security facts change.
