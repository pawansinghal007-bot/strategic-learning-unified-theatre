# strategic-learning-unified-theatre — Launch Brief

## What It Is
strategic-learning-unified-theatre is a local-first development intelligence product that captures useful signals from AI-assisted work and turns them into memory, continuity, and better future prompting.

## Why Now
Developers increasingly work across ChatGPT, Claude, Gemini, Perplexity, editors, terminals, and local tools.
The workflow is powerful but fragmented.
This product connects those fragments into a continuous learning loop.

## The Problem
AI-assisted development is fast, but each session often starts from scratch.
Context gets lost, mistakes repeat, and useful outputs are hard to reuse systematically.

## The Product Story
The product records meaningful workflow signals, stores them locally in a structured experience database, and uses that history to improve the next session.
Over time, the workflow becomes more consistent, more informed, and more reusable.

## Core Features
- Browser conversation capture
- VS Code passive learning
- Structured experience database
- Prompt improvement based on prior context
- Mistake tracking and rubric reuse
- Session handoff continuity
- Training export readiness

## Target Audience
- AI-native developers
- Technical founders
- Researchers who work across many AI sessions
- Privacy-conscious power users

## Why It Wins
- Local-first control
- Cross-session continuity
- Structured memory instead of ad hoc notes
- Practical workflow improvement before full model training

## Current Proof Points
- Core capture and ingestion flows implemented
- Experience database operational
- Prompt and handoff workflows in place
- Training export path implemented
- Documentation hygiene workflow established

## Current Limitation
The local fine-tuning path is not yet production-ready because paired training data is still too sparse.

## Near-Term Plan
- Improve paired capture quality
- Strengthen product docs
- Tighten onboarding
- Validate repeatable user workflows
- Prepare clearer launch messaging

## Demo Narrative
A user works with an online LLM, writes code in VS Code, hits errors, resolves them, and closes the session.
The next session starts with context already preserved, instead of rebuilding everything from memory.

## Ask
Position the product first as a local memory and continuity layer for AI-assisted development, then expand toward self-improving local intelligence.
