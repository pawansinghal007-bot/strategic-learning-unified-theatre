# Audit Trail

## Purpose
Document how to preserve and review durable project history.

## Current approach
- Use the database for active sprint state and decision records.
- Keep master instructions for stable rules and completed sprint summaries.
- Use `docs/archive/` for historical documentation that is no longer current.

## Audit guidance
- Record durable changes in the appropriate curated docs folder at sprint close.
- Avoid adding temporary sprint notes or task lists to curated docs.
- Use archive documents for superseded material.
