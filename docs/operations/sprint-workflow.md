# Sprint Workflow

## Purpose
Describe the sprint workflow without keeping active sprint state in curated docs.

## Current process
- Active sprint state is recorded in the database.
- Handoff snapshots summarize the state for the next session.
- Master instructions are updated at sprint close with durable changes only.

## What belongs in docs
- Durable process guidance.
- Where to find the DB-based state.
- Sprint-close expectations for documentation maintenance.

## What does not belong in docs
- Active sprint task lists.
- Temporary blockers or daily status updates.
- Drafts of handoff notes.
